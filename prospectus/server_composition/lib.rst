===================
サーバーの構成
===================


概要
====

従来のサーバー(2017年7月時点)では主にTCP/IP通信のサーバーが使われている。

現在のサーバーには以下の問題があると考えている。

1. ソケット通信を実装したクライアントを言語ごとに作成しなければならないため、一つの言語あたりのコストがかかること。

2. ルールに変更があった時クライアント側にも対応が必要であるため、維持管理が求められること。

3. ルールがサーバー側のソースコードにハードコーディングされている形になっていること。

4. データの送信方式がJSON形式しかないこと。

そこで上に挙げた問題を解決するためのサーバーの構成を提案する。
